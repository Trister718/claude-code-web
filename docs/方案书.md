# Claude Code Web Chat — 方案书

## 一、项目目标

在本地搭建一个 Web 聊天界面，连接当前使用的 Claude Code（通过 Anthropic API / DeepSeek 代理），实现：

- 在浏览器中与 Claude 对话，替代终端 CLI 交互
- 完美渲染 Markdown 回复（代码高亮、表格、数学公式等）
- **一比一复刻 Claude.ai 网页版的 UI 风格**
- **展示完整中间过程** — 像 CLI 一样看到 thinking、工具调用、文件操作等每一步

### 技术选型

| 层 | 技术 | 理由 |
|---|---|---|
| 前端 | 纯 HTML/CSS/JS (无框架) | 零依赖、零构建、一个文件即可运行 |
| Markdown | marked.js + highlight.js + KaTeX | 对标 claude.ai 的渲染效果 |
| 后端 | Node.js + Express | 轻量、支持流式 SSE、JS 全栈一致 |
| Agent 引擎 | 自实现 Agent Loop | 让 Web 体验 = CLI 体验 |
| 工具系统 | Read/Write/Edit/Bash/Grep/Glob/WebSearch | 对等 Claude Code CLI 的工具能力 |
| API 通信 | Anthropic Messages API (SSE + tool_use) | 直接使用现有的 API Key 和 Base URL |
| 存储 | 本地 JSON 文件 | 对话持久化，简单可靠 |

## 六、SSE 事件流设计 (后端→前端)

### 6.1 事件类型一览

```
事件流示例 (用户发送 "帮我创建一个 hello.txt")：

event: agent_start
data: {"conversationId":"conv_1","messageId":"msg_xxx"}

event: thinking_delta
data: {"text":"我来帮用户创建 hello.txt 文件，先用 Write 工具..."}

event: tool_call
data: {"id":"toolu_001","name":"Write","input":{"file_path":"/path/hello.txt","content":"Hello World"}}

event: tool_result
data: {"tool_use_id":"toolu_001","content":"File created successfully at /path/hello.txt"}

event: thinking_delta
data: {"text":"文件已创建成功，现在回复用户确认。"}

event: thinking_done
data: {}

event: text_delta
data: {"text":"已创建 hello.txt"}

event: text_delta
data: {"text":"文件，内容是 \"Hello World\"。"}

event: text_done
data: {}

event: agent_done
data: {"usage":{"input_tokens":150,"output_tokens":80}}
```

### 6.2 完整事件类型表

| 事件名 | 方向 | 说明 | 前端行为 |
|---|---|---|---|
| `agent_start` | srv→ui | Agent Loop 开始 | 显示加载指示器 |
| `thinking_delta` | srv→ui | Thinking 文本流 | 追加到 Thinking 区块 |
| `thinking_done` | srv→ui | Thinking 结束 | 折叠 Thinking 区块 |
| `tool_call` | srv→ui | 工具调用开始 | 新建工具调用区块，显示工具名+参数 |
| `tool_result` | srv→ui | 工具执行结果 | 填充工具结果内容 (可展开) |
| `text_delta` | srv→ui | 最终回复文本流 | 追加到助手消息 Markdown 渲染 |
| `text_done` | srv→ui | 回复文本结束 | Markdown 最终渲染 |
| `agent_error` | srv→ui | Agent Loop 出错 | 显示错误信息 |
| `agent_done` | srv→ui | Agent Loop 完成 | 隐藏加载器，显示用量统计 |
| `stop` | ui→srv | 用户点击停止 | 中断 Agent Loop |

### 6.3 对话管理 API

```
GET    /api/conversations          -> 获取所有对话列表
GET    /api/conversations/:id      -> 获取单个对话详情 (含中间结果)
POST   /api/conversations          -> 创建新对话
DELETE /api/conversations/:id      -> 删除对话
POST   /api/agent/stream           -> 发送消息并开始 Agent Loop (SSE)
POST   /api/agent/stop/:id         -> 停止正在运行的 Agent
```


### 10.2 Agent Loop 详细流程

```
用户输入消息
    |
    v
将用户消息加入 messages[]
    |
    v
+-> 调用 Anthropic Messages API (stream=true, tools=[...])
|       |
|       v
|   收到 streaming chunk:
|       |
|       +-- content_block_start (type=thinking)  --> SSE: thinking_delta
|       +-- content_block_delta (thinking)       --> SSE: thinking_delta
|       +-- content_block_start (type=tool_use)  --> SSE: tool_call
|       |       |
|       |       v
|       |   执行工具 (Read/Write/Bash/...)
|       |       |
|       |       v
|       |   将 tool_result 加入 messages[]    --> SSE: tool_result
|       |       |
|       |       +-- 继续循环 ---+
|       |                       |
|       +-- content_block_delta (text)         --> SSE: text_delta
|       +-- message_stop                       --> SSE: agent_done
|       |                       |
|       v                       |
|   完成，返回最终回复            |
+------- 如果还有 tool_use ------+
```

1. **API Key 安全**：服务仅监听 `127.0.0.1` (localhost)，不暴露到局域网/公网
2. **CORS**：需要在服务端正确配置 CORS 头，允许本地前端访问
3. **流式响应**：需使用 `fetch` + `ReadableStream` 而非 `EventSource` (POST + 自定义 Header)
4. **DeepSeek 兼容性**：使用 DeepSeek 代理时，需确认其支持 tool_use 和 thinking 功能；如仅支持基础 Messages API，则降级为纯文本对话模式
5. **工具安全**：Bash 工具需要沙箱/确认机制，防止危险命令执行
6. **不引入数据库**：使用 JSON 文件存储对话，适合本地单用户场景
7. **中间结果存储**：对话 JSON 中需存储完整的中间结果 (thinking、tool_calls、tool_results)，以便刷新后恢复完整上下文
